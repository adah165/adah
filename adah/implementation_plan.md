# خطة عمل: إنشاء صفحة سياسة الخصوصية (Privacy Policy) لتوثيق تطبيق Google OAuth

توضيح لخطوات إنشاء صفحة سياسة الخصوصية باللغة العربية تحت مسار `/privacy` لتلبية متطلبات جوجل للتحقق من التطبيق (Google OAuth Verification).

## متطلبات التحقق المطلوبة من جوجل (Google Verification Requirements)
لتجاوز شاشة التحذير والحصول على موافقة جوجل لاستخدام صلاحية `https://www.googleapis.com/auth/adwords` (Sensitive Scope)، يجب أن توضح صفحة سياسة الخصوصية صراحةً:
1. **البيانات التي نجمعها:** معرفات إعلانات جوجل (Customer ID, Campaign ID)، بيانات النقرات (GCLID)، وعناوين الـ IP الخاصة بالزوار لكشف الاحتيال.
2. **الغرض من جمع البيانات:** إدارة وتحسين الحملات الإعلانية، وحظر النقرات الاحتيالية واستبعاد الـ IPs المشبوهة تلقائياً.
3. **مشاركة البيانات:** التأكيد المطلق على **عدم مشاركة أو بيع أو نقل** أي بيانات تم الحصول عليها عبر Google APIs إلى أي طرف ثالث أو استخدامها لأغراض إعلانية خارج نطاق إدارة حملات المستخدم.
4. **حماية البيانات وسرية المعلومات:** تشفير البيانات أثناء النقل والتخزين.
5. **التحكم وحذف البيانات:** إمكانية إلغاء الربط وحذف البيانات بالكامل بناءً على طلب المستخدم عبر البريد الإلكتروني المعتمد `support@adah.sa`.

## خطة التحقق والتمحيص (Verification Plan)

### التحقق التلقائي
- تشغيل أمر البناء والتحقق من صحة الكود وسلامة التصدير:
  ```bash
  npm run build
  ```

### التحقق اليدوي (للمستخدم)
1. **تصفح الصفحة:** بعد تشغيل الخادم محلياً، قم بزيارة الرابط [http://localhost:3000/privacy](http://localhost:3000/privacy) والتأكد من:
   - أن التصميم متناسق وجذاب ومريح للقرّاء ومراجعي جوجل.
   - دعم التجاوب مع مختلف شاشات الهواتف والأجهزة اللوحية والمكتبية.
   - التحقق من شمولية صياغة البنود باللغة العربية الفصحى القانونية السليمة.
2. **طباعة الصفحة:** تجربة الضغط على زر طباعة الصفحة (أو Ctrl+P) للتأكد من تنسيق الطباعة الأنيق والخالي من العناصر الجانبية غير الضرورية.
3. **تطابق الشروط:** مطابقة بنود الصفحة مع الشروط التي تفرضها جوجل في سياسة خدمات واجهة برمجة التطبيقات (API User Data Policy).
