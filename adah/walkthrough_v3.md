## 🚀 خطوتك التالية في GitHub Desktop لرفع التحديثات

اتبع هذه الخطوات البسيطة لرفع التعديلات إلى المستودع الخاص بك (GitHub):

1. **افتح برنامج GitHub Desktop**.
2. ستجد التعديلات الجديدة تظهر تلقائياً في القائمة اليسرى (الملفات المعدلة: `Logo.tsx`, `package.json`, `signup/route.ts`, `verify/route.ts`, `.env`).
3. في مربع الـ **Summary (مطلوب)** في الأسفل اليسار، اكتب عنواناً للتحديث مثل:
   ```text
   تحديث الشعار لـ logo.png وتهيئة إعدادات البريد وأمر بناء Vercel
   ```
4. اضغط على زر **Commit to main** (أو اسم الفرع الحالي الخاص بك).
5. بعد انتهاء الـ Commit، اضغط على زر **Push origin** في الأعلى لرفع الملفات إلى GitHub.

## ⚙️ إعدادات Vercel لضمان عمل الموقع بالكامل (حل خطأ 404)

بما أن المشروع يقع داخل مجلد فرعي باسم `adah` وليس في جذر المستودع مباشرة، يجب ضبط الإعدادات التالية في لوحة تحكم Vercel:

### 1. ضبط المجلد الرئيسي (Root Directory):
1. اذهب إلى لوحة تحكم **Vercel** وافتح مشروعك.
2. اذهب إلى تبويب **Settings** (الإعدادات) في الأعلى.
3. تحت تبويب **General**، ابحث عن خيار **Root Directory**.
4. اضغط على **Edit** واكتب أو اختر المجلد: `adah` ثم اضغط **Save**.

### 2. إضافة متغيرات البيئة (Environment Variables):
في صفحة الإعدادات ذاتها في Vercel، اذهب إلى تبويب **Environment Variables** وقم بإضافة المتغيرات التالية (انسخ قيمها من ملف الـ `.env` المحلي الخاص بك):
- `DATABASE_URL`: رابط الاتصال بقاعدة البيانات (Supabase PostgreSQL).
- `NEXTAUTH_SECRET`: مفتاح التشفير السري لـ NextAuth.
- `NEXTAUTH_URL`: رابط موقعك المرفوع على Vercel (مثال: `https://your-project.vercel.app`).
- `RESEND_API_KEY`: مفتاح الـ API الخاص بـ Resend لإرسال الرسائل.
- `EMAIL_FROM`: البريد الإلكتروني الذي سيرسل الرسائل (مثال: `أداة <onboarding@resend.dev>`).
- `GOOGLE_CLIENT_ID` و `GOOGLE_CLIENT_SECRET`: لتسجيل الدخول بحساب Google.
- `GOOGLE_ADS_DEVELOPER_TOKEN` و `GOOGLE_ADS_CUSTOMER_ID`: للاتصال بـ Google Ads.

### 3. إعادة بناء ونشر الموقع:
بمجرد قيامك بالضغط على **Push** في GitHub Desktop، سيقوم Vercel تلقائياً باكتشاف التحديثات وبناء المشروع بناءً على المجلد الجديد `adah` وسيعمل الموقع بالكامل دون أي أخطاء 404!
